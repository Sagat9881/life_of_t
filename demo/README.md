# Life of T — Demo Module

Interactive demo of the *Life of T* game engine. Runs as a self-contained
Spring Boot application that generates pixel-art sprite atlases at startup,
serves an animated home-room page, and exposes REST endpoints backed by real
domain entities.

---

## Quick Start

```bash
# From the repository root (builds all modules first):
mvn install -DskipTests

# Run the demo:
mvn spring-boot:run -pl demo

# Open in browser:
open http://localhost:8080
```

---

## What You See

The home page (`/`) renders:

| Element | Asset | Animation |
|---------|-------|-----------|
| Home-room background | `home_room_bg.png` | Static (1 frame) |
| Tanya (player character) | `tanya_idle.png` | 6-frame idle cycle @ 8 fps |
| Sam the dog | `sam_idle.png` | 4-frame tail-wag @ 6 fps |
| Bed (furniture) | `bed_static.png` | Static (1 frame) |

All sprites use **horizontal-strip atlases** — every frame is laid out
left-to-right in a single PNG. JavaScript advances the
`background-position-x` each tick. No GIFs, no anti-aliasing, max 50 frames
per animation.

The side panel shows live data polled every 5 seconds from the REST API:
- **Stat bars** — energy, health, stress, mood, money, self-esteem
- **Active quests** — with per-step progress
- **Relationships** — closeness hearts for Husband and Father

## REST API

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/demo/status` | Full game-state snapshot (character, quests, time, relationships) |
| `GET` | `/api/demo/quest-log` | Active quests array |
| `GET` | `/api/demo/characters` | Character info map |
| `GET` | `/api/demo/relationships` | NPC relationship values |
| `GET` | `/api/assets` | List of generated asset metadata |
| `GET` | `/api/assets/{id}.png` | Serve generated PNG sprite atlas |
| `POST` | `/api/demo/shutdown` | Graceful shutdown (used by automation) |

## Asset Generation

Assets are generated by `DemoAssetService` on startup using Java's
`BufferedImage` API — no external tools required. The output directory
defaults to `${java.io.tmpdir}/life-of-t-assets` and is registered as an
additional static-resource location (`/generated-assets/**`).

XML visual-specs under `docs/prompts/` are the authoritative design source for
a future AI-art pipeline:

```
docs/prompts/
  locations/home_room/visual-specs.xml
  characters/tanya/visual-specs.xml
  pets/sam_dog/visual-specs.xml
  furniture/bed/visual-specs.xml
```

## Tests

### Smoke test
```bash
mvn test -pl demo -Dtest=DemoApplicationTests
```

### Quest-chain E2E
```bash
mvn test -pl demo -Dtest=QuestChainE2ETest
```
Exercises `QuestLog`, `Quest`, `QuestStepState` directly (Java domain objects)
and asserts narrative coherence via the REST API.

### Conflict-resolution E2E
```bash
mvn test -pl demo -Dtest=ConflictResolutionE2ETest
```
Tests the full `Conflict` lifecycle: BREWING → ESCALATION → CLIMAX →
RESOLUTION, with `BaseConflictTactics` (SURRENDER, ASSERT, LISTEN, COMPROMISE,
AVOID) and relationship mutations.

### Game narrative E2E (Day 1 simulation)
```bash
mvn test -pl demo -Dtest=GameNarrativeE2ETest
```
Multi-step simulation of Tanya's Day 1: wake up, self-care, work,
husband date, Sam playtime, end-of-day decay, multi-day conflict trigger.

### Browser E2E (requires Docker)
```bash
mvn test -pl demo -Dtest=DemoBrowserE2ETest
```
Uses Testcontainers `BrowserWebDriverContainer` (headless Chrome) to
verify the page renders in a real browser. Skipped automatically if Docker
is unavailable.

## Architecture

```
demo/
  src/main/java/ru/lifegame/demo/
    DemoApplication.java          # Spring Boot entry point + startup listener
    config/
      DemoConfiguration.java      # WebMvcConfigurer: static resource handler
    controller/
      DemoController.java         # REST: /api/demo/**
      AssetController.java        # REST: /api/assets/**
    service/
      DemoGameService.java        # Domain session: PlayerCharacter, QuestLog, Relationships
      DemoAssetService.java       # PNG atlas generation (BufferedImage, no anti-aliasing)
    dto/
      DemoDtos.java               # Java 21 records: GameStateDto, QuestSummaryDto, etc.
  src/main/resources/
    application.yml
    static/
      index.html                  # Single-page animated room (inline CSS + JS)
  src/test/java/ru/lifegame/demo/
    DemoApplicationTests.java     # Smoke tests
    e2e/
      QuestChainE2ETest.java      # Quest lifecycle + REST integration
      ConflictResolutionE2ETest.java  # Conflict domain tests
      GameNarrativeE2ETest.java   # Full Day 1 narrative simulation
      DemoBrowserE2ETest.java     # Testcontainers + Selenium Chrome
```

### Key design decisions

- **No Lombok** — all DTOs are Java 21 records; service state uses
  `AtomicReference<>` for thread safety without synchronization.
- **No preview features** — `pom.xml` overrides the parent's
  `--enable-preview` with an empty `<compilerArgs/>`.
- **Clean Architecture** — domain objects from `ru.lifegame.backend.domain.*`
  are used directly; the demo layer only adds Spring wiring and DTO mapping.
- **Horizontal-strip atlas only** — every animation is a single PNG with
  frames left-to-right. The JavaScript animator advances
  `background-position-x` by `frameWidth` px per tick.
- **Max 50 frames** — enforced by `DemoAssetService.generate()` with
  `Math.min(spec.frameCount(), 50)`.

## CI

GitHub Actions workflow: `.github/workflows/demo-ci.yml`

- Triggers on push/PR to `master` when `demo/**`,
  `backend/**`, or `asset-generator/**` change.
- Builds all modules, runs demo tests (browser E2E excluded by default),
  packages the fat JAR, uploads it as an artifact.
- Browser E2E enabled by setting the `BROWSER_E2E_ENABLED` repository
  variable to `true`.
