# ADR-002: generation-complete.sentinel как сигнал завершения генерации

| Поле | Значение |
|------|----------|
| **Статус** | Принято |
| **Дата** | 2026-04-03 |
| **Связанные** | ADR-001-visual-docs-data-independence |

---

## Контекст

CI-пайплайн (`asset-generation.yml`) ранее ждал появления файла `tanya_idle.png` как
сигнала завершения генерации ассетов. Это нарушало ADR-001: имя конкретной сущности было
захардкожено в инфраструктурном коде CI. Если персонаж Tanya удалялась или переименовывалась,
CI зависал навсегда.

## Решение

В `AssetGeneratorRunner` добавлен метод `writeSentinel(Path dir)`, который пишет файл
`generation-complete.sentinel` как **абсолютно последнее** I/O-действие раннера во всех
режимах (`STANDARD` и `DOCS_PREVIEW`).

### Контракт файла

| Атрибут | Значение |
|---------|-----------|
| Имя | `generation-complete.sentinel` |
| Расположение | `ASSET_OUTPUT_DIR` (корень директории с ассетами) |
| Содержимое | Одна строка — ISO-8601 timestamp (`2026-04-03T12:34:56.789Z`) |
| Порядок записи | **Последний файл**, записываемый раннером |
| Присутствие в артефакте | Да — загружается вместе с ассетами для аудита |

### Почему не `sprite-atlas.json`

`sprite-atlas.json` создаётся только для сущностей с анимациями
(`hasCharacterAnims || hasOverlayLayers` в `LayeredAssetGenerator`). Статичные локации
и объекты без анимаций (например, `bed_static.png`) его не создают. Следовательно,
`sprite-atlas.json` не является надёжным «последним файлом» при наличии таких сущностей.

## Последствия

- CI больше не содержит имён сущностей в шагах ожидания и верификации.
- Добавление новых персонажей/локаций в `specs-manifest.xml` не требует изменений в CI.
- Верификация перешла от «проверить список конкретных PNG» к «проверить sentinel + кол-во PNG > 0».
- Детальные проверки размеров атласов делегированы unit-тестам (`DocsPreviewUseCaseTest`,
  будущим тестам `LayeredAssetGeneratorTest`).
- Если sentinel не записан (runner упал до `writeSentinel`), CI фиксирует таймаут с ошибкой.
