# Структура промптов проекта Life of T

## Принцип организации

Промпты организованы вокруг **акторов системы** - каждая сущность (персонаж, локация, объект) имеет свою директорию.

## Навигация

### Быстрый доступ

- **Основной workflow**: `_core/visual-generation-workflow.xml`
- **Главная героиня**: `characters/tanya/visual-specs.xml`
- **Главная локация**: `locations/home/generation-specs.xml`

### Шаблоны

Используй `_*-template.xml` файлы как основу для создания новых сущностей того же типа.

## Принципы именования

1. **Директории**: `snake_case` для имён сущностей
2. **Файлы**: `kebab-case.xml` для спецификаций
3. **Префикс `_`**: для системных/шаблонных файлов
4. **Суффикс `-specs.xml`**: для визуальных спецификаций
5. **Суффикс `-config.xml`**: для конфигураций

## Связи между промптами

- Все промпты персонажей ссылаются на `_core/visual-generation-workflow.xml`
- Все промпты локаций ссылаются на паттерн генерации трёх планов
- Все анимации следуют правилу минимум 20-25 кадров
