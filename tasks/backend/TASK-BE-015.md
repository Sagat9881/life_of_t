## Описание

Отсутствует подтверждение DDD-структуры пакета `ru.lifegame.backend`. По результатам baseline-аудита 30.03.2026 невозможно верифицировать, что код соответствует архитектурным слоям Domain / Application / Infrastructure / Presentation, зафиксированным в `java-developer-skill.md`.

Необходимо провести аудит существующей структуры пакетов и зафиксировать результат в техническом документе, указав соответствие / отклонения от DDD-шаблона.

## Выходные артефакты

### Обязательные разделы документа:
1. **Эталон** — целевая DDD-структура по `java-developer-skill.md`
2. **Текущее состояние** — реальная карта пакетов `ru.lifegame.backend`

4. **GAP-список** — пакеты/классы, не вписывающиеся в DDD
5. **Рекомендации** — что рефакторить, какие задачи поставить
