## Контекст

Проект реализует документационный сайт, визуально отображающий персонажей, локации и UI-компоненты игры «Life of T». Для каждой не-abstract сущности из `specs-manifest.xml` необходима документационная визуальная спека в `docs/visual-specs/`. Именно эти спеки читает генератор ассетов в режиме `docs-preview` для создания PNG-превью и метаданных карточек сайта.

**Зона ответственности Нарратора**: содержательное заполнение спек — описание персонажа/локации для документации, выбор анимации для превью, указание действия при клике. Технический формат XML фиксирован схемой и **не изменяется Нарратором**.

Принцип **ADR-001** (data-independence): спеки не должны содержать хардкода технических параметров — они берутся из игровых спек генератором ассетов автоматически.

## Что нужно сделать

Для каждой не-abstract сущности из `specs-manifest.xml` создать файл:

```
docs/visual-specs/{entityType}/{entityId}/visual-specs.xml
```

Использовать `docs/visual-specs/_template/visual-specs.xml` как основу (копировать, не редактировать шаблон).

### В каждой спеке заполнить:

1. **`<meta>`**:
   - `entity-type` — тип сущности (`character`, `location`, `furniture`, `ui`)
   - `entity-name` — читаемое имя на русском
   - `version` — начальное значение `1.0`

2. **`<layers>`**:
   - Минимум один слой с описанием визуального плана **словами** (не код, не HTML).
   - Описание должно помочь зрителю документации понять, как выглядит персонаж/локация.

3. **`<color-palette>`** *(опционально)*:
   - Основные цвета персонажа или локации — для справочника документации.

4. **`<docs-rendering>`**:
   - `<caption>` — краткое название на **русском** для карточки сайта. Примеры: «Татьяна — главная героиня», «Кухня — место утреннего кофе».
   - `<preview-animation>` — id анимации из игровой спеки сущности, подходящий для превью (как правило `idle`; для локаций — `ambient` или аналог).
   - `<click-action>` — что показывать при клике по карточке. Допустимые значения: `show-animation` | `open-spec` | `both`.

5. **`<animations>`**:
   - Описание ключевых анимаций для документации. Не все — только те, что важны для понимания персонажа/локации.
   - Каждая запись: `animation-id` + краткое текстовое описание (1–2 предложения).

Выполнять в порядке приоритета:

| Очередь | Группа сущностей | Количество |
|---|---|---|
| 1 (первая) | `characters/` | 12 персонажей |
| 2 (вторая) | `locations/` | 12 локаций |
| 3 (третья) | `furniture/` и `ui/` группы | по количеству из манифеста |

> Точный список сущностей каждой очереди — из `specs-manifest.xml` (фильтр: `abstract != true`).

## Ограничения для Нарратора

Согласно `narrantor-skill.md`:

- ❌ Не изменять технический формат XML — только содержательные поля (`<caption>`, `<preview-animation>`, `<click-action>`, `<layers>`, `<color-palette>`, `<animations>`).
- ❌ Не создавать спеки в `game-content/` — только в `docs/visual-specs/`.
- ❌ Не определять технические параметры (`fps`, `frame-width`, `bit-depth`) — они берутся из игровых спек генератором.
- ❌ `<caption>` — строго не длиннее **60 символов**. Никакого лирического текста в XML.
- ❌ `<click-action>` — строго из допустимого набора: `show-animation` | `open-spec` | `both`.
- ❌ Не трогать файлы в `game-content/` или `asset-generator/`.

## Критерии приёмки

- [ ] Для каждой не-abstract сущности из `specs-manifest.xml` создан файл `docs/visual-specs/{entityType}/{entityId}/visual-specs.xml`.
- [ ] Все файлы проходят валидацию: `xmllint --noout docs/visual-specs/**/*.xml` → без ошибок.
- [ ] Все `<click-action>` — одно из: `show-animation`, `open-spec`, `both`.
- [ ] Все `<caption>` — не длиннее 60 символов.
- [ ] CI-шаг `validate-docs-specs` (TASK-BE-DOC-003) проходит без ошибок после добавления спек.
- [ ] Нарратор не трогал файлы в `game-content/` или `asset-generator/` (проверка через `git diff --name-only`).
- [ ] Ни один файл спеки не содержит технических параметров, которые должны браться из игровых спек (fps, frame-width, bit-depth).
