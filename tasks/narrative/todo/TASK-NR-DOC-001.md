## Описание

Нарратор создаёт документ `docs/visual-specs/visual-brief.md` — описание
визуального тона и ощущения документационного сайта для JavaScript Developer.

Цель: JS Developer должен понимать, какую атмосферу создавать в CSS/UI сайта,
не нарушая принцип data independence.

## Содержание брифа

1. **Общий тон** — как должен ощущаться сайт (тепло, уют, пиксельная эстетика).
2. **Цветовая гамма** — рекомендуемые акцентные цвета (не конкретные hex, а настроение).
3. **Типографика** — пиксельный шрифт или системный?
4. **Карточки** — как должны выглядеть карточки персонажей vs локаций vs мебели.
5. **Анимации** — насколько активны, какое настроение передают.

## Критерии приёмки

- [ ] Документ создан в `docs/visual-specs/visual-brief.md`.
- [ ] Содержит все 5 разделов.
- [ ] Нет технических деталей (JSON, CSS, JS) — только нарративное описание.
- [ ] Соответствует тону игры «Life of T» (светлый ромком, уют, тепло).
